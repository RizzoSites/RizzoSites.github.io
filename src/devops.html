How we work

Project support according to DevOps methodology is a one of the main areas of ​​our activity. With Effective interaction with your team we shall make the following steps:

Audit and Design

Key-turn CI/CD concept design, from analysis of development process to defining the new pipeline methodology.

Applying Microservices

Transferring projects to Microservice architecture with the division of the application into independent blocks and simplifying the work.

Creating development environment

Deploying Kubernetes cluster and configuring the application stack in accordance with the developed methodology.

Support and monitoring

Support of CI/CD processes for built infrastructure throughout the project cycle.

What will I get?

- A full description of the IaC (Infrastructure as Code) methodology. Chef will create rules for deploying a common part of the infrastructure for applications, which include MySQL, MongoDB, RabbitMQ, Cassandra and Ceph, a Kubernetes cluster (PaaS), and virtual machine and hypervisor settings

- GitLab installation, in which the Continuous Integration and Continuous Delivery workflows will be worked out

- Ready-to-go virtual machines, in your private or public cloud

- Technical support 24 × 7 × 365: continuous support of development processes

- Statistics and monitoring system, backup

- Guarantees of fault tolerance and scalability of the architecture of the created infrastructure.
